С сайта отправляется форма -> приходит на php код тот реквест
Далее проверки с crm:
1. Контакта нет с номером в реквесте -> делается сделка и контакт в нужной воронке(Микрорайон или ЖК)
2. Контакт есть, но все сделки по нему в Закрыто не реализовано или в Успешно реализовано (это состояния у воронок) -> тогда создается только сделка на этот контакт
3. Если контакт есть, но сделка является активной -> тогда сделка не создается и контакт тоже не создается.

При всех 3 случаях проверки происходят такие действия с карточкой в амо:


- Из формы в поле "Город объекта" записывается - РНД\КРД\СТВ в зависимости от выбранного города в форме

- В зависимости от выбранного ЖК в форме, сделка создается в нужной воронке (воронки в амо подписаны по типу "РНД ЖК Вересаево" или "РНД ЖК Левобережье")

- Номер телефона клиента из формы записывается в поле телефона контакта

- Имя клиента из формы в название контакта (если контакт уже создан, то изменять не надо)

- Переписка с клиентом и Комментарий от специалиста из формы приходят в виде 1 или 2 примечаний, главное чтобы было красиво и чительно

- Ставится тег на сделку - konkurenty.gk-usi.ru

- В поле "Источник сделки" записать значение - "Лид по конкурентам"

- Падает новая сделка в этап Лид, если сделка уже есть активная, то переносить не надо

- Ставится задача типа "Связаться", с текстом "Связаться с клиентом", срок выполнения До конца дня
